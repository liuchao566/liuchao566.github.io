java之session持久化

一般情况下，浏览器关闭，会话结束，下次打开浏览器，服务器会创建新的session，存储的信息会丢失，那以前的session在哪里？
以前的session还在服务器内存中，tomcat默认的session存活时间是30分钟，可以在web.xml中配置。
当使用session时，服务端会向客户端写一个名为JSESSIONID的cookie，该cookie默认有效时间是0。
每次访问，客户端都会带着JSESSIONID，而服务端底层通过JSESSIONID找到session。
